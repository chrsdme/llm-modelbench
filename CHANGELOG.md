## 1.0.0-rc9

### Context truth and model-card telemetry

- Rejects GPU-VRAM-only measured slopes once dynamic offload begins. Process PSS/RSS is used when available; system-wide RAM remains diagnostic-only. Estimates are monotonic and cannot fall below observed residency.
- Records per-needle-tier request/server timing, prompt/decode throughput, GPU/CPU telemetry, system RAM/swap, Ollama process RSS/PSS/swap, model residency from `/api/ps`, response-shape warnings, and 64k operating-readiness fields.
- Adds configurable long-context operating targets: 64k by default, 10 tok/s preferred decode speed, and 3 tok/s critical floor.

### Current-first KV repair

- Guarded needle repair now attempts the current/default KV configuration before any sudo preflight, service discovery, or systemd mutation.
- q8/q4 are bounded fallbacks only for unresolved current-KV work. Build/runtime-scoped Flash-Attention incompatibility evidence is persisted and suppresses pointless repeated quantized-KV attempts until identity changes.
- q4 is skipped when q8 proves the shared quantized-V-cache Flash-Attention prerequisite is unavailable.

### Real-time repair watch

- Writes `repair_link.json` before child generation starts.
- Merges the live child `status.json` into the parent repair campaign view. Needle depth, target context, state, and latest speed/resource evidence update while the task is running.

### Capability routing

- Actual scored runs functionally probe lanes by default; use `--no-auto-probe` only for deliberate metadata-only routing.
- Capability decisions combine operator profiles, Ollama metadata, conservative name hints, functional responses, and task-equivalent FIM probing.
- A responding-but-wrong probe routes the scored task as a quality test; only definitive unsupported-build evidence excludes a capability lane. Transient or ambiguous probe failures are withheld and logged, not mislabeled unavailable.

### Ranking semantics

- Consumes build-specific capability-unavailable evidence and bounded recovery exhaustion without fabricating positive scores.
- Adds long-context operating profiles to ranking/model-card payloads. Rankings V3 visual integration remains deferred until these semantics are validated on the real host.

## 1.0.0-rc1

Initial public release candidate.

### Benchmark lanes

- Deterministic text, coding, JSON, git, file-operation, and web checks.
- Deterministic agentic JSON tool-action lane; the benchmark never executes
  external tools itself.
- Capability-first vision routing for OCR and PDF tasks, including local
  image fixtures where a task declares `image_path`.
- Embedding retrieval lane: recall@1 and MRR, with `embed_model` provenance
  recorded on every row.
- Long-context needle probes with explicit coverage and safety gates.

### Reports and artifacts

- HTML, Markdown, CSV, JSON, routing, prune, clone, and regression reports.
- Raw result rows, summary metadata, filters, digest-backed model identities,
  and review-pack export.
- Retrieval diagnostics artifacts with case-level ranks (target rank,
  top-3 IDs, target/nearest-distractor similarity, margin, pass@1) for
  instrumented rows; older rows are labelled partial rather than
  reconstructed.
- Interactive and plain watch layouts, including a TTY-safe interactive mode.

### Operator tools

- Digest-keyed coverage ledger and advisory coverage gaps (never schedules a
  run itself).
- Read-only dossier: a cross-run composite over covered, non-stale
  categories, renormalized to what actually ran. Missing or stale coverage is
  never scored as zero.
- Report-time category-weight overrides that write a separate report copy
  without mutating raw artifacts.
- Repeatability report and empirical noise-band guidance
  (`diff --noise-band`).
- Sensitivity planning/reporting, VRAM-skip simulation, and a read-only
  routing server over existing summaries.

### Safety boundaries

- No model deletion or automatic run scheduling.
- Read-only tools never benchmark or refresh model data.
- Public fixtures must be synthetic; see
  [docs/PRIVACY_FIXTURES.md](docs/PRIVACY_FIXTURES.md).

### Known limitations

- Retrieval adversarial cases currently number a small set; treat close
  rankings between embedding models as provisional pending a larger case set.
- `gaps` is intended to be verified against a live Ollama server before being
  fully trusted in a given environment; it is exercised in `--mock` mode and
  against synthetic ledgers in the test suite.
